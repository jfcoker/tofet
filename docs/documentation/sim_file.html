1.5.3. The ‘sim’ file¶

The .sim file contains all the simulation variables. The format of the file is:

KEYWORD   VALUE   [VALUE_2]   [VALUE_3]

where KEYWORD relates to one of the simulation variables, and VALUE is its value. Multiple values can be given when submitting batch jobs with tft_run_batch.

Depending on the simulation you are running, some variables are necessary, whilst others are optional. Different simulations are discussed in the section on Examples, and a summary of all variables is given in appendix~ref{appen:sim}.

section{Simulation exampleslabel{sec:use}} Examples of several different simulations are given here, in rough order of increasing complexity. If you don’t see an example that relates to your problem, but suspect that ToFeT might be able to handle it, email me~cite{me}: you might be right.

Please note: textbf{all examples provided are highly simplistic!} In particular, to ensure good convergence, you need to ensure that your morphology is sufficiently large, and your simulation sufficiently well converged.
