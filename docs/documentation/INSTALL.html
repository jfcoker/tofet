1.4. Installation¶

1.4.1. Prerequisites¶

1.4.1.1. C++ compiler¶

The only requisite to run the core code (contained in /trunk/source) is a C++ compiler.

1.4.1.2. GNU Scientific Library (GSL)¶

By default, ToFeT uses the GSL to generate random numbers. However, if you don’t have this installed, an alternative generator is included in the package; see the section: Random number generators.

1.4.1.3. Command-line interface, Python, and Libraries¶

A lot of the functionality that ToFeT offers for the extraction and analysis of data is provided by Python scripts run from the command line. A Python installation of 2.5 or more recent is necessary. For some scripts, including tft_calc_ecp.py and tft_rotate_coords.py, SciPy and NumPy are also necessary.

1.4.2. Building ToFeT¶

  1. Download a ***.tar.gz (or, if you’re feeling bold, check it out of the repository directly).
  2. Unpack: tar -zxvf ToFeT_**.tar.gz
  3. Edit the Makefile (see comments within)
  4. Cross your fingers
  5. Type: make

It is highly recommended that you allow your compiler to make optimisations. ToFeT was developed using the GCC, version 4.3. Any other compiler is likely to give warnings - please let me know of these and I’ll make the necessary clean-ups.

If you don’t have the GSL installed type: make randomB instead of make.

The Makefile generates two executables, tft and tft_occ. The former is the most general, the latter is used only when you want to track the time that each molecule is occupied (see below).

1.4.3. Testing ToFeT¶

A test suite is provided for the GSL random number generator in trunk/examples/GSL_randomGenerator/test. Make sure that tft and tft_occ are in your path, and run the tests by typing:

python tofetTest.py
