3.2. Simulation parameters¶

alpha¶

(tof or regenerate modes only).

The geometric scaling of time bins.

See also section Time in ToFeT.

converged¶
(1,0) (fet mode only). If 1, set the hopper density as converged immediately. Make sure to feed the simulation the correct distribution of hoppers in a occ file as an option to protect_me.
cyclesForConverence¶
(1,0) (fet only). The current must be within tol for this many cycles for the current to be deemed converged. Defaults to 15. See also movesCycle
dielectric¶
(regenerate and fet modes only). Whenever hopperInteractions are enabled (by default for fet), specify the dielectric constant.
fieldZ¶
The field along the z axis in qV/Ang, where q=$pm$1 for holes / electrons. protect_me always expects charges to drift along the z axis. In a normal setup the generators are at small z’s and the collectors are at large z’s. In this case fieldZ must be negative so that charges move in a positive direction along z. textbf{Note:} because q is contained within this definition of the field, no adjustment to fieldZ has to be made between hole and electron simulations: both drift along z for negative fieldZ’s.
deltaTime¶

(tof or regenerate modes only). The width of the first time bin used to store transient current events (see section~ref{sec:tof:configuring}).

See also section Time in ToFeT.

hoppers¶
(tof or regenerate mode only). The number of hoppers (charges).
hopperInteractions¶
(1, 0) Turns on carrier-carrier Coulombic interactions. Currently only enabled for the fet and regenerate modes.
maxRuns¶
(tof or regenerate modes only). The maximum number of runs. If the mobility hasn’t converged within maxRuns, the simulation will be truncated.
maxTime¶

The maximum time of simulation (s). Simulations will be truncated at maxTime unless they have already converged.

See also section Time in ToFeT.

mode¶
(tof, regenerate, fet) By default protect_me simulates the time-of-flight experiment in which charges are removed from the simulation whenever they are collected. However, protect_me can also run in the regenerate mode where, every time a charge is collected, another is immediately regenerate mode.
movesCycle¶
(fet mode only). Convergence is checked every movesCycle Monte Carlo steps. Defaults to 2e4. See also cyclesForConvergence.
printOccupation¶
(1, 0) If 1, print the occupation probabilities of each molecule (see section~ref{sec:use}). textbf{Note:} to get occupation probabilities you also need to use the binary protect_meOccupation.
printVertices¶
(1,0) If 1, print the details of each molecule.
printEdges¶
(1,0) If 1, print the details of all intermolecular interactions. Warning, not fully tested.
reorg¶
The reorganisation energy (eV)
siteEnergies¶
(1,0) By default protect_me expects $Delta E$ in your edge file, unless you are in fet mode or have activated hopperInteractions. However, if you are providing site energies (E) in your .xyz file instead, you must specify siteEnergies 1 in your .sim file. Site energies must be provided for fet and for regenerate simulations when hopperInteractions are on.
temp¶
The temperature (K)
tol¶
Stop the simulation when the fractional change in the mobility~/~current is between 1-tol and 1+tol. track (1,0) Track each hop of each hopper. Must be used in conjunction with the executable protect_meOccupation.
Vds¶
(fet mode only). The source-drain voltage (qV), where q=$pm$1 for holes / electrons. As for fieldZ, no adjustment has to be made to Vds to switch from simulating holes to simulating electrons since q is contained in the units; in both cases both drift along z for negative Vds.
Vg¶
(fet mode only). The gate voltage (qV), where q=$pm$1 for holes / electrons. As for Vds, no adjustment has to be made for holes / electrons; in both cases charges are drawn into the channel by positive Vg.
verbosity¶
(high, low) By default the verbosity of the output is low; set to high if you want more runtime information.
